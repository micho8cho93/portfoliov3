# Application Layer

The application layer is where user-facing rules live: naming, requesting, and protecting content.

<InternetLayerVisual layer="application" analogyLabel="Restaurant front desk: names, menu orders, and sealed envelopes." />

## Real-World Analogy

Think of a restaurant front desk:

- `DNS` is the host stand translating a restaurant name into a street number.
- `HTTP` is how you place and receive your order.
- `TLS` is a sealed envelope so others cannot read your order on the way.

## Components In This Layer (From the PDF Definitions)

- `DNS`: Converts readable names into numeric destination addresses.
- `HTTP`: Defines request and response message format for web resources.
- `TLS`: Adds encryption and integrity checks to protect application data.

## What The Loop Visual Shows

The cards represent browser and server. Packets move in both directions for request and response, DNS flips from name to number, and the lock pulse represents TLS protection.
